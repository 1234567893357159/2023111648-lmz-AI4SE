7.3 实验背景
经过实验二至实验六，已经分别完成了传统机器学习、深度学习以及大语言模型代
码审查模型的构建，并针对AI生成代码进行了性能分析与优化。然而，这些模型主要
以脚本程序的形式运行，距离真实的软件开发环境仍存在一定差距。
VSCode 作为目前最广泛使用的软件开发工具之一，提供了完善的插件开发接口，
使开发者能够将智能分析能力直接集成到软件开发过程中，实现开发、代码审查和反馈
的一体化。
本实验将在前六个实验实现的代码审查模型基础上，开发一个VSCode智能代码审
查插件，使开发者能够在编写代码过程中实时调用代码审查模型，预测PullRequest是
否能够被合并，并自动生成代码审查意见，从而提高软件开发效率。
7.4 实验原理
7.4.1 VSCode 插件架构
VSCode 插件通常包括以下几个组成部分：
VSCode
↓
Extension API
↓
插件逻辑
↓
代码审查模型
↓
结果展示
插件通过Extension API 获取编辑器中的代码信息，并调用后端模型完成智能代码
审查。
7.4.2 插件功能设计
本实验设计的VSCode 插件主要包括以下功能：
• 获取当前编辑文件；
• 获取代码修改内容；
• 调用代码审查模型；
• 展示Merge Prediction 结果；

• 展示自动生成的代码审查意见；

7.4.3 模型集成方式
为了便于模型部署，本实验采用前后端分离方式：
• VSCode 插件负责用户交互；
• Python 后端负责调用机器学习、深度学习及大语言模型；
• 前后端通过HTTP接口进行通信。
这种架构能够方便模型更新，同时保证插件具有良好的扩展性。
7.4.4 结果可视化
为了帮助开发者快速理解代码质量，本实验设计以下展示方式：
• Merge Prediction 结果；
• 代码审查意见列表；
• 风险等级提示；
• 高亮显示存在问题的代码位置；
• 模型推理时间及调用信息。
