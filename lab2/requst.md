2.3 实验背景
实验一完成了代码审查数据集的构建，为代码审查模型训练提供了基础数据。然而，
原始的Pull Request 数据不能直接用于机器学习模型训练，需要首先完成数据预处理和
特征工程。
传统机器学习方法通常依赖人工设计的特征，通过提取代码结构、代码复杂度、文
本信息以及开发过程中的统计信息，将代码修改表示为固定长度的特征向量，再利用分
类模型判断Pull Request 是否能够成功合并（Merge）。
虽然近年来深度学习和大语言模型已经广泛应用于代码理解任务，但传统机器学习
方法仍具有训练成本低、模型可解释性强以及训练速度快等优点，因此仍然是代码审查
领域的重要基线方法。本实验将基于实验一构建的数据集，训练传统机器学习模型，为
后续深度学习模型提供性能比较基准。

2.4.2 代码中间表示
程序代码可以转换为不同的中间表示形式，从不同角度描述程序语义。
• 抽象语法树（Abstract Syntax Tree，AST）：描述程序语法结构；

• 控制流图（Control Flow Graph，CFG）：描述程序执行流程；
• Diff 信息：描述代码修改内容；
• Commit Message：描述开发者修改意图。
不同表示能够提供互补的信息，共同支持后续特征提取。
2.4.3 特征工程
本实验主要构建以下几类特征：
• 代码结构特征（AST节点数量、CFG节点数量等）；
• 代码修改特征（新增代码行数、删除代码行数、修改文件数量等）；
• 文本特征（Title、Body、Commit Message 长度等）；
• 审查特征（Reviewer 数量、Comment 数量等）。
上述特征经过归一化处理后构成机器学习模型输入。
2.4.4 传统机器学习模型
本实验主要采用以下两种经典模型：
1. 支持向量机（Support Vector Machine，SVM）
SVM 通过寻找最优分类超平面对不同类别进行划分，适用于中小规模数据集。
2. 随机森林（Random Forest）
随机森林由多个决策树组成，通过集成学习提高模型泛化能力，并能够分析各特
征的重要程度。
