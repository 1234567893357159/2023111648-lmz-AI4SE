算法竞赛纯 C 语言避坑指南（从 C++ 转 C 重点注意事项）
竞赛只能写 .c 文件，用 gcc 编译，不能用任何 C++ 特性，对比你熟悉的 C++ 逐条区分，分模块讲竞赛高频坑。
一、头文件：彻底忘掉 bits/stdc++.h
C没有万能头，不能写 #include <bits/stdc++.h>，直接编译报错
竞赛常用 C 头文件清单，按需复制：
c
运行
#include <stdio.h>    // 输入输出必用
#include <stdlib.h>   // malloc/free、abs、rand
#include <string.h>   // strlen、memset、memcpy、strcmp
#include <math.h>     // sqrt、log、pow、fabs
#include <ctype.h>    // isdigit、isalpha
#include <time.h>     // 随机数种子
#include <stdbool.h>  // bool、true/false（C99支持，大部分竞赛编译器开启）
#include <limits.h>   // INT_MAX、INT_MIN
坑：C++ 的 <iostream>、<vector>、<queue>、<algorithm>、<string> C 全部不存在。
二、输入输出：抛弃 cin/cout，只用 scanf/printf
1. 为什么不能用 cin/cout
.c 文件不识别 C++ 流，编译直接报错。全程只用 scanf、printf。
2. 常用格式符（竞赛必考，记牢）
表格
类型	scanf 读入	printf 输出
int	%d	%d
long long	%lld	%lld
unsigned int	%u	%u
double	%lf	%f
char 字符	%c	%c
字符串 char []	%s	%s
3. 高频踩坑
long long 千万不能用 %d，会读错数值；
double 输入必须 %lf，输出用 %f；
读字符时 scanf("%c", &ch) 会读入换行空格，解决方案：scanf(" %c", &ch)（前面加空格）；
输出大量数据时，C 的 printf 比关闭同步的 cin 还快，不用担心超时。
4. 快速读入替代（大数据卡常用）
没有 ios::sync_with_stdio(false);，只能手写快读函数：
c
运行
int read(){
    int x=0,f=1;char ch=getchar();
    while(ch<'0'||ch>'9'){if(ch=='-')f=-1;ch=getchar();}
    while(ch>='0'&&ch<='9'){x=x*10+ch-'0';ch=getchar();}
    return x*f;
}
三、数据结构：没有 STL，全部手动实现（最大难点）
你 C++ 常用的 vector、queue、stack、map、set、string 在 C 里全部不存在，必须手写。
1. 动态数组（替代 vector<int>）
C 没有自动扩容 vector，两种方案：
预估最大范围，开静态数组（竞赛最常用，简单高效）
c
运行
#define MAXN 100005
int a[MAXN];
动态内存 malloc/realloc（不确定数据范围时用）
c
运行
int *arr = (int*)malloc(sizeof(int)*n);
arr = (int*)realloc(arr, sizeof(int)*new_size); // 扩容
free(arr); // 使用完必须释放，不然内存泄漏（竞赛一般无所谓，但规范要写）
2. 字符串（替代 C++ string）
C 只有 char[] 字符数组，不能直接 s1=s2 赋值，不能 s.size()
获取长度：int len = strlen(s);
复制字符串：strcpy(a,b);
拼接：strcat(a,b);
比较：strcmp(a,b)，相等返回 0
注意：数组要多开一位存结束符 \0
3. 队列、栈、链表：手动数组模拟最优
竞赛不推荐指针链表（容易 RE、慢），用数组模拟：
数组模拟队列示例
c
运行
#define MAXQ 100005
int q[MAXQ];
int l=0,r=0;
// 入队
q[r++]=x;
// 出队
l++;
// 判断空
if(l>=r)
栈更简单，单数组 + 栈顶指针即可。
4. map/set（哈希、离散化）
C 无内置红黑树，两种处理：
数据范围小：数组标记；
数据大、离散：手写离散化 + 二分查找；
哈希：手写哈希表（开放寻址 / 链地址）。
四、语法核心区别（C vs C++，超多语法不兼容）
1. 变量定义位置（C89 硬性限制）
很多 OJ 默认 C89 标准：变量必须写在代码块最开头，不能中间定义
错误：
c
运行
for(int i=1;i<=n;i++) // C89不允许循环里定义int i
{
    int x=read(); // 循环中间定义变量报错
}
正确写法：
c
运行
int i,x;
for(i=1;i<=n;i++)
{
    x=read();
}
部分 OJ 支持 C99 可以中间定义，但竞赛稳妥起见统一放块开头。
2. 函数：不能函数内嵌套函数
C++ 可以函数嵌套 lambda，C 完全不行，所有函数写在全局域。
3. bool 类型
需要手动引入头文件 #include <stdbool.h> 才能用 bool true false；
不引入只能用 int 代替布尔（0 假，非 0 真）。
4. 强制类型转换
C 转换写法：(long long)x，没有 long long(x) 这种 C++ 构造式转换。
5. 没有引用 &
C++ void swap(int &a,int &b) 引用传参 C 不存在；
交换变量只能传指针：
c
运行
void swap(int *a,int *b){
    int t=*a;
    *a=*b;*b=*t;
}
// 调用 swap(&x,&y);
6. 结构体无直接赋值外的便捷操作
C 结构体不能直接 struct Node a={1,2},b; b=a; 部分编译器支持，但不能直接比较；
没有结构体运算符重载、没有类、没有封装；
结构体定义简写：
c
运行
typedef struct{
    int x,y;
}Node;
Node a; // 不用写struct Node
7. 无 namespace、无 using
完全不用管，C 没有命名空间概念。
五、常用算法函数：没有 algorithm 库，全部手写
C++ 自带 sort、swap、max、min、lower_bound 全部不能用
排序：C 自带 qsort（stdlib.h），或手写快排 / 归并
qsort 示例（int 数组）：
c
运行
int cmp(const void *a,const void *b){
    return *(int*)a - *(int*)b; // 升序
}
qsort(a,n,sizeof(int),cmp);
二分查找：自己手写二分，没有 lower_bound/upper_bound
max/min：自己写宏或函数
c
运行
#define max(a,b) ((a)>(b)?(a):(b))
#define min(a,b) ((a)<(b)?(a):(b))
六、内存与指针高频坑（竞赛最容易 RE）
静态数组开太大（全局 / 局部区别）
全局数组：存在堆，开 1e6、1e7 没问题；
main 内部局部数组：栈空间很小，超过 1e5 直接运行时错误 RE；超大数组一定要定义全局。
malloc 返回值判断：内存不足会返回 NULL，竞赛数据大时可能爆内存；
memset 只适合清零 0 或置 -1，不能用来赋值 1、0x3f 之类非 0/-1 值（按字节赋值）。
七、刷题适配建议（快速从 C++ 转 C）
模板全部重写一套 C 版本
快读、快速输出、数组模拟队列栈、qsort 比较函数、二分、高精度、图论邻接表全部改成 C；
放弃所有 STL 思维，优先静态数组（竞赛最快、最简单，少用 malloc）；
输入输出只练 scanf/printf，不要混用；
做题时强制不用任何 C++ 语法，写完自查：
有没有 cin/cout？
有没有 vector/string/map？
有没有循环内定义变量？
有没有引用 &、lambda、类？
数据类型优先 long long：乘法容易爆 int，C 没有自动 long 转换，溢出 WA 很难查。
八、竞赛加分小技巧
宏定义简化代码：
c
运行
#define MAXN 100005
#define INF 0x3f3f3f3f
全局数组替代动态内存，减少 RE；
多组输入统一用 while(scanf("%d",&n)!=EOF)；
浮点数比较不要直接 ==，用 fabs (a-b)<1e-8 判断相等。
九、总结最容易丢分的关键点
误用 %d 输出 long long → WA；
C89 块中间定义变量 → 编译错误；
局部开大数组栈溢出 → RE；
想用 STL 容器、bits 万能头 → 编译失败；
字符串直接赋值、比较不用 strcmp → 逻辑错误；
忘记指针传参，函数内交换无效；
想用 sort，忘记只能用 qsort 手写比较函数。
