3.3 实验背景
实验二采用传统机器学习方法完成了代码审查任务，通过人工设计代码结构特征、

本实验将在实验二构建的数据集基础上，利用Code2Vec 和 CodeBERT 模型学习

审查意见生成（Review Comment Generation）两个典型任务，为后续基于大语言模型
的代码审查实验奠定基础。
3.4 实验原理
3.4.1 深度学习代码审查流程
深度学习代码审查通常包括以下几个步骤：
代码输入
↓
代码表示学习
↓
深度神经网络编码
↓
任务微调
↓
Merge Prediction / Review Comment Generation
与传统机器学习不同，深度学习能够自动学习代码语义表示，无需人工设计大量特
征。
3.4.2 Code2Vec 模型
Code2Vec 是一种基于抽象语法树路径（ASTPaths）的代码表示学习模型。该模型
首先从程序抽象语法树中提取大量路径，再利用注意力机制学习不同路径的重要程度，
最终生成整个代码片段的向量表示。
Code2Vec 能够有效捕获程序的语法结构信息，适用于代码分类、代码克隆检测以
及代码审查等任务。
3.4.3 CodeBERT 模型
CodeBERT 是一种面向源代码与自然语言联合训练的预训练模型。模型利用Trans
former 结构，通过海量开源代码进行预训练，能够学习程序语义和自然语言描述之间的
对应关系。

3.4.4 代码审查任务
本实验主要完成以下两个任务：

根据Pull Request 修改内容预测该Pull Request 最终是否能够被项目接受并合并。

建议。

3.6 实验内容
本实验需要完成以下任务：
1. 从实验二构建的数据集中读取人类编写代码对应的数据；
2. 利用Code2Vec 提取代码语义表示；
3. 利用CodeBERT 学习代码表示；
4. 微调CodeBERT 完成 Merge Prediction 任务；
5. 微调CodeBERT 完成 Review Comment Generation 任务；
6. 比较不同深度学习模型的性能。
3.7 实验步骤
3.7.1 步骤一：数据准备
读取实验二处理完成的数据集，并划分训练集、验证集和测试集。
3.7.2 步骤二：Code2Vec 代码表示学习
利用Code2Vec 模型生成代码向量表示，完成代码编码，并保存得到的向量表示结
果。
3.7.3 步骤三：CodeBERT 模型微调
利用预训练 CodeBERT 模型完成模型微调，分别针对 Merge Prediction 任务和
Review Comment Generation 任务训练模型。
3.7.4 步骤四：模型预测
利用训练完成的模型，对测试集中的PullRequest 进行预测，包括：
• 是否Merge 预测；
• 自动生成代码审查意见。
