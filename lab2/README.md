# 实验二：基于机器学习的人类编写代码的代码审查

## 实验概述

本实验在 lab1 的基础上，从 5 个大型开源项目中筛选人类编写的 Pull Request，提取其修改前后的完整源代码文件，并利用 tree-sitter 对代码进行 AST（抽象语法树）和 CFG（控制流图）解析，构建结构化的代码特征数据集。最终基于这些特征，使用 SVM 和 Random Forest 两种机器学习模型，预测 PR 是否会被合并（Merge）。

## 实验目标

1. 从 lab1 的原始数据中筛选人类编写的 PR（排除 Bot 和 AI 生成代码）
2. 通过 GitHub API 下载每个 PR 修改前后的完整源代码文件
3. 使用 tree-sitter 解析代码，生成 AST 和 CFG 中间表示
4. 从 PR 元信息、AST、CFG 中提取多维特征
5. 使用 SVM 和 Random Forest 训练 PR 合并预测模型，并对比评估

## 实验流程

### 步骤一：PR 筛选

从 lab1 爬取的 5 个项目的 PR 数据中，筛选出人类编写的 PR：

- **Bot 排除**：通过作者名关键词（如 `bot`、`dependabot`、`renovate` 等）排除自动化提交
- **AI 检测**：三级检测策略
  1. 关键词匹配：检查 commit message、PR body、title、labels 中是否包含 AI 生成标识（如 `co-authored-by: copilot`、`generated by ai` 等）
  2. 启发式规则：大文件单次提交 + 短 commit message；模板化 body 结构
- 筛选结果分别保存到 `data/raw/human/` 和 `data/raw/non_human/`

### 步骤二：代码提取与中间表示生成

对每个筛选出的人类 PR：

1. **代码下载**：通过 GitHub API 获取 PR 修改前后的完整源代码文件，支持 Go、Python、JavaScript、TypeScript、Java、C、C++ 等多种语言
2. **AST 生成**：使用 tree-sitter 解析源代码，提取控制流节点（if/for/while/return 等）信息
3. **CFG 生成**：识别函数定义，对每个函数构建控制流图（节点为语句，边为控制流转移），计算圈复杂度等指标

### 步骤三：特征提取

从三个维度提取特征：

- **基础特征**：文件数、增删行数、标题/正文长度
- **AST 特征**：各类控制流节点数量（sum/mean），总节点数
- **CFG 特征**：各类节点/边数量（sum/mean），函数数量，平均每函数节点数/边数，圈复杂度

所有特征按 before/after 并列提取，最终汇总为 `human_features.csv`。

### 步骤四：模型训练

使用 SVM（支持向量机）和 Random Forest（随机森林）两种模型：

- 数据划分：80% 训练集，20% 测试集（stratify 分层抽样）
- 特征标准化：StandardScaler
- 超参数优化：GridSearchCV（3 折交叉验证）
  - SVM：RBF 核，搜索 `C`、`gamma` 参数
  - Random Forest：搜索 `n_estimators`、`max_depth`、`min_samples_split` 参数

### 步骤五：模型评估

- 评估指标：Accuracy、Precision、Recall、F1-score、ROC-AUC
- 可视化：混淆矩阵、ROC 曲线对比、特征重要性 Top 20
- 输出评估报告到 `data/results/evaluation_report.txt`

## 快速开始

### 1. 配置 GitHub Token

打开 `config.py`，将 `GITHUB_TOKEN` 替换为你的 GitHub Personal Access Token：

```python
GITHUB_TOKEN = "your_github_token_here"
```

Token 生成方式：GitHub Settings → Developer Settings → Personal Access Tokens → Tokens (classic)，无需任何权限（public repo 只读即可）。

> **注意**：Token 仅用于提高 API 频率限制（5000 次/小时 vs 60 次/小时），确保数据下载过程不被限流。

### 2. 配置网络代理（国内环境）

如果在中国大陆运行，可能需要配置代理。在 `config.py` 中设置：

```python
HTTP_PROXY = "http://127.0.0.1:7890"
HTTPS_PROXY = "http://127.0.0.1:7890"
```

留空则自动使用环境变量中的代理设置。

### 3. 运行主流程

打开 `main.ipynb`，按顺序执行每个 Cell：

1. **步骤一**：筛选人类 PR → 运行 `pr_filter.py`
2. **步骤二**：下载代码 + 生成 AST/CFG → 运行 `code_extractor.py` 和 `ast_cfg_generator.py`
3. **步骤三**：特征提取 → 运行 `feature_extractor.py`
4. **步骤四**：模型训练 → 运行 `model_trainer.py` 的 `train_svm()` 和 `train_rf()`
5. **步骤五**：模型评估 → 运行 `model_trainer.py` 的 `evaluate_both()`

也可以直接运行对应的 Python 模块：

```bash
# 步骤一：筛选 PR
python pr_filter.py

# 步骤二：提取代码
python code_extractor.py

# 步骤二：生成 AST 和 CFG
python ast_cfg_generator.py

# 步骤三：特征提取
python feature_extractor.py

# 步骤四 & 五：训练与评估
python model_trainer.py
```

### 4. 测试 tree-sitter 解析能力

运行测试脚本，验证各语言解析是否正常：

```bash
python test_language_parsing.py
```

## 关键技术

- **tree-sitter**：增量式语法解析器，支持 Go、Python、JavaScript、TypeScript、Java、C、C++ 等语言，用于生成 AST 和构建 CFG
- **networkx**：图结构库，用于构建和存储控制流图（CFG）
- **scikit-learn**：机器学习库，使用 SVM（RBF 核）和 Random Forest 进行 PR 合并预测
- **GridSearchCV**：网格搜索 + 交叉验证，自动寻找最优超参数
- **圈复杂度（Cyclomatic Complexity）**：基于 CFG 计算，公式 `E - N + 2P`，衡量代码复杂度
