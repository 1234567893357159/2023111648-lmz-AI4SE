4.2 实验目的
完成本实验后，学生应能够：
1. 理解大语言模型在代码审查任务中的应用原理。
2. 掌握Prompt Engineering 的基本思想与设计方法。
3. 理解不同代码上下文对大语言模型性能的影响。
4. 能够利用大语言模型完成MergePrediction 任务。
5. 能够利用大语言模型自动生成代码审查意见。
6. 能够分析不同Prompt 设计和上下文信息对代码审查结果的影响。
4.3 实验背景
实验三采用深度学习模型完成了代码表示学习和代码审查任务，但模型通常需要大
量标注数据进行训练，并且只能针对固定任务进行优化。近年来，大语言模型（Large
Language Models，LLMs）在代码理解、代码生成和软件工程任务中展现出了优异性能，
使代码审查逐渐由传统监督学习转向基于提示词的推理模式。
不同于CodeBERT 等需要重新训练或微调的模型，大语言模型能够利用已有知识
直接完成代码审查任务。然而，大语言模型的性能高度依赖输入信息的组织方式。合理
选择代码上下文以及设计高质量Prompt，能够显著提高代码审查质量。
本实验将在实验三构建的数据集基础上，调用开源或商用大语言模型，通过设计不
同的代码上下文和提示词，完成MergePrediction和Review Comment Generation 两个
任务，并分析不同设计策略对模型性能的影响。

4.4 实验原理
4.4.1 基于大语言模型的代码审查流程
大语言模型代码审查通常包括以下几个步骤：
Pull Request
↓
构建代码上下文
↓
Prompt 设计
↓
调用大语言模型
↓
Merge Prediction / Review Comment Generation
与深度学习模型相比，大语言模型无需重新训练即可完成代码审查任务。
4.4.2 代码上下文
代码审查不仅依赖修改代码本身，还需要结合丰富的软件工程上下文。本实验主要
考虑以下几类上下文：
• Pull Request 描述；
• Commit Message；
• 修改前后的Diff；
• 修改文件内容；
• 历史代码片段；
• 历史代码审查意见。
不同上下文能够为模型提供不同层次的软件工程信息，从而影响最终审查结果。
4.4.3 Prompt Engineering
Prompt Engineering 是影响大语言模型性能的重要因素。
本实验主要探索以下Prompt设计方式：
• Zero-shot Prompt；
• Few-shot Prompt；

• Chain-of-Thought Prompt；
• Role-based Prompt。
不同Prompt 能够引导模型采用不同的推理方式完成代码审查任务。
4.4.4 代码审查任务
本实验主要完成以下两个任务：
1. Merge Prediction
根据Pull Request 修改内容及相关上下文预测是否能够成功Merge。
2. Review Comment Generation
根据Pull Request 生成完整、合理的代码审查意见，并分析不同 Prompt 设计对
生成质量的影响。

4.6 实验内容
本实验需要完成以下任务：
1. 读取实验三构建的人类代码数据集；
2. 构建不同粒度的代码上下文；
3. 设计不同类型的Prompt；
4. 调用大语言模型完成MergePrediction 任务；
5. 调用大语言模型生成代码审查意见；
6. 比较不同上下文和Prompt 设计对模型性能的影响。
4.7 实验步骤
4.7.1 步骤一：数据准备
读取实验三中的测试数据，并构建代码审查任务所需的数据输入。
4.7.2 步骤二：构建代码上下文
分别构建以下不同类型的上下文：
• 仅包含Diff；
• Diff + Pull Request 描述；
• Diff + Commit Message；
• Diff + 等其他信息。
4.7.3 步骤三：Prompt 设计
针对每种上下文分别设计：
• Zero-shot Prompt；
• Few-shot Prompt；
• Chain-of-Thought Prompt；
• Role-based Prompt。

4.7.4 步骤四：模型推理
调用大语言模型分别完成：
• Merge Prediction；
• Review Comment Generation。
记录模型输出及推理时间。
4.7.5 步骤五：结果分析
统计不同上下文和Prompt设计下模型性能，包括但不限于：
• Accuracy；
• Precision；
• Recall；
• F1-score；
• BLEU；
• ROUGE；
• 推理时间。
比较不同Prompt 设计和上下文组合的优缺点。

4.9 实验思考
请结合实验过程回答以下问题：
1. 为什么代码上下文会影响大语言模型的推理能力？
2. Few-shot Prompt 相比 Zero-shot Prompt 有哪些优势？
3. 哪类上下文对Merge Prediction 贡献最大？为什么？
4. 不同Prompt 设计为什么会生成不同质量的代码审查意见？
5. 大语言模型相比实验三中的深度学习模型有哪些优势与不足？
4.10 实验拓展
完成基础实验后，可进一步尝试：
• 探索Repository 级上下文、跨文件上下文对代码审查性能的影响；
• 尝试Self-Consistency、Self-Reflection 等高级 Prompt 策略；
• 比较不同大语言模型（如Qwen、DeepSeek、GPT等）在代码审查任务上的表现，
为后续AI生成代码的代码审查实验提供基准。
